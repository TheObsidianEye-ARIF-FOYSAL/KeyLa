Cryptography

The parts worth checking

Every primitive here is libsodium's, used the way it was designed to be used. Nothing invented, nothing improvised.

Argon2id
Stretches your master password with a memory-hard KDF and a domain-separated salt, so a stolen vault resists offline cracking.
derive
XChaCha20-Poly1305
Authenticated encryption. Envelope-wraps the vault key and seals every stored field — tampering is detected, not just decrypted into garbage.
seal
SQLCipher
The database file itself is encrypted with a random, device-bound key held in the Keystore — separate from your master password.
at rest
Server auth secret
The server never receives your master password. A separate secret is derived from it via a domain-separated Argon2id call before anything is sent.
zero-knowledge
Flow

Three steps, then it disappears

STEP 01

Set a master password

Sign in with your number, then choose one strong master password. A strength meter guides you, and you leave with a recovery kit — write it down, because nobody can reset it for you.

STEP 02

Fill the vault

Add logins by hand or generate them on the spot. Each field is sealed before it touches storage, so the database never holds a readable secret.

STEP 03

Unlock and forget it

Biometric unlock, a health score when you want to tidy up, and an encrypted backup whenever you choose to push one.

⚠

The browser demo is a preview, not a vault

SQLCipher, Keystore-held keys, biometric unlock, and screenshot blocking are native capabilities with no browser equivalent. In the demo, storage falls back to IndexedDB — fields are still sealed by the app's own encryption, but the layers underneath are gone. Use the Android app for real passwords.
