Features

Everything a vault should do

From generating a strong password to spotting the weak ones you already have.

🗄️

Encrypted Vault

Every credential lives in a SQLCipher-encrypted database, with each field individually sealed before it's written to disk.

🎲

Password Generator

CSPRNG-backed random passwords and memorable passphrases, with length, character sets, and ambiguous-character controls.

📊

Password Health

A scored dashboard that flags weak, reused, and stale passwords so you know exactly what to fix first.

🔒

Auto-lock & Screen Privacy

Locks after 60 seconds of inactivity or when backgrounded, blocks screenshots on Android, and blurs on iOS.

👆

Biometric Unlock

Unlock with fingerprint or face, gated by the Keychain/Keystore — with your master password always as the fallback.

☁️

Encrypted Backup

Sync an already-encrypted vault blob to your own server. It leaves your device as ciphertext and comes back the same way.

Security

What actually protects your data

No hand-rolled crypto — Keyla uses libsodium primitives the way they were designed to be used.

🧮 Argon2id

Your master password is stretched with a memory-hard KDF and a domain-separated salt. Argon2id

🛡️ XChaCha20-Poly1305

Authenticated encryption envelope-wraps the vault key and every stored field. AEAD

🗝️ SQLCipher

The database file itself is encrypted with a random, device-bound key held in the Keystore.

🙈 Zero-knowledge server

The backend stores a bcrypt account hash and an opaque blob. It never sees a plaintext credential.

How it works

Three steps to a sealed vault

1

Create your master password

Sign in with your phone number, then set one strong master password. A strength meter guides you, and you get a recovery kit to store safely.

2

Add your credentials

Save logins with a title, username, password, and notes — or generate a fresh password on the spot. Everything is encrypted before it touches storage.

3

Unlock, check, and back up

Unlock with biometrics, review your password health score, and push an encrypted backup to your server whenever you want.

⚠️ About the browser preview

The web build is a UI preview only. The encrypted SQLCipher database, Keystore-backed key storage, biometric unlock, and screenshot blocking are native-only capabilities that have no browser equivalent — so the in-browser version cannot store a real vault. Use the Android app for actual password storage.
